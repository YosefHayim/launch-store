Выпуск приложения — это не только сборка: настройка подписи, конфигурация [App Store Connect](https://developer.apple.com/app-store-connect/) / [Play Console](https://play.google.com/console),
встроенные покупки, метаданные карточки приложения, загрузка и последующие обновления «по воздуху» (OTA).
EAS собирает и отправляет — всё остальное разбросано по порталам Apple и Google и нескольким инструментам.
Launch собирает **весь релиз** в один локальный декларативный рабочий процесс: он подготавливает вашу подпись,
сверяет ваши товары в магазине, генерирует нативный проект, собирает и подписывает бинарник, сообщает реальный
размер загрузки на конкретное устройство, сохраняет артефакт и загружает его в тестовый трек — на вашем
собственном оборудовании, с ключами, которые остаются в вашем локальном keychain. Подпись iOS требует Mac;
если его у вас нет, Launch собирает на облачном Mac в **вашем собственном** аккаунте AWS либо передаёт работу
Expo EAS — см. [Сборка без Mac](#сборка-без-mac).

> **Впервые здесь?** Запустите `launch demo` для 60-секундного симулированного обзора всего конвейера — без
> настройки, без сборки, без аккаунта. Он также проигрывается автоматически при первом запуске `launch`.

## Зачем нужен Launch

Сборка никогда не была самой сложной частью — сложен весь релиз вокруг неё. Launch владеет всей этой
поверхностью, локально и с открытым исходным кодом:

- **Весь релиз — один рабочий процесс.** Подпись, товары в магазине, сборка, проверка размера, загрузка и
  OTA-обновления исходят из одного `launch.config.ts` и горстки команд — а не из дюжины дашбордов и CI-сниппетов.
- **Настройка магазина как код.** `launch sync` сверяет встроенные покупки, подписки, цены и возможности
  (capabilities) с App Store Connect из одного типизированного `launch.config.ts`; ещё дюжина команд покрывает
  Game Center, Wallet, App Clips, внутриигровые события, A/B-эксперименты, территории и весь каталог Google
  Play — плюс `launch metadata` для карточки приложения. Это те части, которые EAS оставляет вам кликать вручную.
- **$0 за вычисления, неограниченные сборки.** EAS тарифицирует по сборкам, ограничивает бесплатный тариф
  45-минутным таймаутом и стоит **$19–$199/мес** плюс перерасход на платных планах. Launch собирает на вашей
  собственной машине — без счётчика, без очереди, без таймаута.
- **Ваши ключи остаются локальными.** Ваш сертификат распространения, ключ API App Store Connect и ключ
  загрузки Android остаются в вашем keychain ОС; Launch лишь отправляет CSR в Apple. (Сборка без Mac — единственное
  исключение, см. ниже.)
- **Никакой привязки к вендору, никогда.** Лицензия MIT, основа на `fastlane`, Gradle и собственных API
  платформ, с подключаемыми провайдерами хранилища/учётных данных/сборки/отправки. Ничего проприетарного, с чего
  пришлось бы мигрировать позже.
- **Обучает по ходу работы.** Добавьте `--explain` к любой команде — или запустите `launch demo` — чтобы
  развернуть каждый шаг (CSR, профиль подготовки, TestFlight, трек Play, группа подписки) на понятном языке.

## Что такое Launch — и чем он не является

**Launch — это** сквозной инструмент релиза: он владеет всем путём от исходного кода до магазина — сборка,
подпись кода, проверка размера, настройка магазина как код, загрузка, публичный релиз и OTA-обновления — как для
iOS, так и для Android, на оборудовании, которым вы владеете.

**Launch — это не** просто SDK для App Store Connect или ASC MCP-сервер. Они оборачивают лишь часть API Apple;
Launch управляет всем релизом сразу в Apple **и** Google, с подписью, сборкой и OTA-обновлениями, которых обёртка
над API не касается. Если вам нужен self-hosted Expo EAS — а не просто клиент API — это Launch.

## Возможности

**Настройка и проверка**

- **Конфигурация в один шаг.** `launch init` обнаруживает ваше приложение (или приложения) — включая монорепозиторий
  `apps/*` — и пишет прокомментированный `launch.config.ts` (+ стартовый `.env.example`). Он только пишет конфиг; он
  никогда не трогает учётные данные или нативный проект.
- **Ключи в вашем keychain.** `launch creds set-key` находит `AuthKey_*.p8` в `~/Downloads`, проверяет его в Apple
  и сохраняет в защищённом хранилище секретов вашей ОС; `creds setup` регистрирует id приложения и создаёт или
  переиспользует ресурсы подписи; многоаккаунтные `creds use/rename/remove` переключают между командами. `creds push-key`
  помещает в хранилище загружаемый-один-раз ключ авторизации APNs (у Apple нет API для его создания) и реэкспортирует
  его по требованию.
- **Секреты, а не открытый текст в `.env`.** `launch secret set <NAME>` хранит секрет сборки в keychain вашей ОС
  (с областью действия на приложение/профиль) и внедряет его в окружение сборки — не давая реальным секретам
  попасть в закоммиченный `.env`; `secret list` (с маскировкой) и `secret rm` дополняют картину.
- **`launch doctor --fix`.** Обнаруживает инструментарий iOS/Android и устанавливает недостающие brew-устанавливаемые
  инструменты по одному согласию (`--yes` пропускает его для CI/агентов), помечает блокеры на стороне магазина —
  отсутствующую запись в App Store Connect, неподписанное соглашение Apple — и проверяет вашу конфигурацию Expo на
  известные подводные камни нативной настройки (неправильный bundle id / package, splash без `backgroundColor`) — тот
  же предполётный контроль, который `launch build` выполняет заранее, так что однострочная ошибка в конфиге падает за
  секунду, а не за сборку. Он также вскрывает две ловушки времени отправки, о которых EAS умалчивает: ответ об
  **экспортном соответствии** (read once from `ios.config.usesNonExemptEncryption`, и с `--fix` отвечается на последней
  сборке через API), и одноразовый ручной опросник **App Privacy** — для которого Apple не предоставляет API, поэтому
  Launch печатает точный чеклист, вместо того чтобы дать ему внезапно заблокировать отправку.
- **Статус подписи с первого взгляда.** `launch setup ios` сообщает о вашей подготовке iOS из конца в конец — активный
  аккаунт, App ID, возможности, сертификат распространения, профиль и зарегистрированные устройства — а с `--provision`
  обеспечивает сертификат + профиль App Store, то же самое, что `launch creds setup`.

**Настройка App Store Connect — как код**

Каждый раздел ниже объявляется в `launch.config.ts` (или в его автономном sidecar-файле `*.config.json`) и сверяется
по принципу «план только для чтения → ваше подтверждение → применение» — идемпотентно, никогда не затрагивая
активную или находящуюся на ревью версию. Это та поверхность, которую EAS целиком оставляет сайту App Store Connect.

- **Товары, цены и карточка приложения.** `launch sync` сверяет ваши встроенные покупки, подписки, возможности и
  **цены** с App Store Connect — и в том же проходе **тексты карточки** по каждой локали (название, подзаголовок,
  описание, ключевые слова, что нового, URL конфиденциальности / поддержки / маркетинга), **скриншоты** и
  **превью приложения** — сразу по всем приложениям.
- **Предложения по подписке.** `launch offers` сверяет коды предложений, а также промо-, ознакомительные и
  возвратные (win-back) предложения, плюс порядок продвигаемых покупок; `launch offers generate-codes/list/deactivate`
  управляют кампаниями кодов предложений из CLI.
- **Атрибуты релиза.** `launch release-config` сверяет **возрастной рейтинг App Store, категории, базовую цену и
  данные App Review** (контакт + демо-аккаунт) с редактируемой версией.
- **Идентичность и права приложения.** `launch game-center` (достижения и таблицы лидеров), `launch wallet`
  (идентификаторы продавца Apple Pay и идентификаторы типов пропусков Wallet), `launch app-clips` (действие карточки
  App Clip + подзаголовок) и `launch eu-distribution` (домены альтернативного распространения в ЕС + ключ подписи
  пакета, для DMA) — те клики по порталу для настройки команды, которые `spaceship` вскрывает, а EAS — нет.
- **Мерчандайзинг и присутствие.** `launch availability` (территории, где продаётся приложение), `launch custom-pages`
  (альтернативные страницы товара), `launch experiments` (A/B-тесты страницы товара), `launch accessibility`
  (метки питательной ценности по доступности), `launch events` (внутриигровые события) и `launch metadata pull/push`
  (полная карточка для **iOS _и_ Android** — `eas metadata` работает только с iOS).

**Настройка Google Play — как код**

- **Товары и подписки Play.** `launch play-products` и `launch play-subscriptions` сверяют ваши встроенные товары и
  подписки Play (базовые планы + предложения) из **того же** каталога `launch.config.ts`, который управляет App Store
  Connect — один источник истины, оба магазина.
- **Треки и отзывы.** `launch play-tracks` показывает статус трека и **продвигает** сборку на трек с выбранным
  процентом выкатки и заметками о релизе (а также читает/задаёт группы тестировщиков); `launch play-reviews` читает
  отзывы клиентов Play и публикует ответы — без открытия Play Console.

**Сборка и публикация — iOS и Android**

- **Одна команда на платформу.** `launch build ios` / `launch build android` выполняет prebuild → подпись →
  проверку размера → загрузку в тестовый трек (TestFlight / внутренний Play) — тот же поток, что запускает EAS.
- **Быстро по умолчанию.** ccache подключается на этапе `pod install`, DerivedData остаётся прогретым, а отпечаток
  нативного графа форсирует чистую сборку только когда ваши нативные зависимости действительно меняются. Правки JS
  пересобираются инкрементально; `--clean` форсирует сборку с нуля.
- **Реальная проверка размера загрузки.** Сообщает фактический размер на конкретное устройство (отчёт App Thinning /
  bundletool) и контролирует его по заданному вами `sizeBudgetMB`.
- **Страховки.** Отказывается загружать сборку для симулятора, `.app` или пустой артефакт; `--dry-run` прогоняет
  весь конвейер без сети, сборки и изменений аккаунта.
- **Осознанный публичный релиз — без портала.** Тестовый трек используется по умолчанию; публичный магазин — это
  отдельная, подтверждаемая `launch release <platform>`. Для iOS она управляет API App Store Connect из конца в
  конец — создать/переиспользовать версию, ответить об экспортном соответствии, прикрепить сборку, написать заметки о
  релизе, выбрать немедленную / запланированную / поэтапную выкатку и отправить на ревью — так что релизу никогда не
  нужен сайт. `launch status [--watch] [--json]` отслеживает ревью (с кодами выхода для CI), а `launch rollout
pause|resume|complete` управляет поэтапной выкаткой. fastlane задействован только для загрузки бинарника.
- **Повторная подпись без пересборки.** `launch build:resign` повторно подписывает сохранённый `.ipa`/`.aab` другими
  учётными данными (новый аккаунт или профиль) прямо из артефакта — без пересборки.
- **Уведомления о завершении.** Блок `notify` пингует webhook Slack/Discord и/или запускает shell-хук, когда сборка
  или отправка завершается — как при успехе, _так и_ при сбое — чтобы автономный/CI-запуск сообщил вам, когда он
  закончен. Паритет с `webhook` из EAS, без хостируемого сервиса.

**Распространение и обновление**

- **Внутреннее распространение.** `launch build <platform> --distribution internal` размещает ad-hoc ссылку на
  установку iOS / `.apk` Android в вашем собственном бакете; регистрируйте тестировщиков через `launch device add <udid>`.
- **Обновления «по воздуху» (OTA).** `launch update` публикует JS/ассет-обновление по **протоколу Expo Updates**,
  на котором `expo-updates` уже говорит — с подписью кода и хостингом в вашем собственном бакете (S3 / R2 / Supabase).
- **Откат неудачного обновления.** `launch updates list/view` показывают историю по каждому каналу; `launch updates
rollback` отменяет неудачный OTA — продвигает заведомо рабочее обновление или возвращает тестировщиков на встроенный
  бандл.

**Управление тестировщиками, командой, отзывами и отчётами — только по ключу API**

- **TestFlight из CLI.** `launch testflight groups/create-group/testers/add/rm` управляет бета-группами и приглашает
  тестировщиков по тому же ключу API App Store Connect — слой управления вокруг загрузки сборки, без пароля Apple ID
  и без 2FA.
- **Отзывы — чтение и ответ.** `launch reviews [list]` читает отзывы клиентов (фильтр по рейтингу / территории);
  `launch reviews reply` публикует или заменяет ответ разработчика, а `launch reviews delete` удаляет его — без
  открытия App Store Connect.
- **Отчёты по продажам, финансам и аналитике.** `launch reports sales/finance/analytics` загружает отчёты Sales &
  Trends, Finance и Analytics из App Store Connect (gzip-сжатый TSV) прямо на вашу машину для скриптинга — те цифры,
  которые EAS никогда не показывает.
- **Команда и доступ.** `launch team list/invite/remove` читает и управляет участниками команды App Store Connect и
  ожидающими приглашениями — приглашайте по email с ролями или отзывайте — по тому же ключу API.
- **Тестировщики песочницы.** `launch sandbox list/clear` перечисляет ваших тестировщиков песочницы StoreKit и
  очищает их историю покупок, чтобы вы могли заново тестировать встроенные покупки с чистого листа.

**Инспекция и отладка**

- **История сборок.** `launch builds list/view/log` читает локальный индекс артефактов — id, размеры на конкретное
  устройство, пути артефактов и сырой лог сборки для любой сборки.
- **Установка и запуск.** `launch run [id|latest]` устанавливает собранный артефакт на подключённое устройство или
  симулятор (`adb`/`bundletool` для Android, `devicectl` для iOS).
- **Объяснение сбоя.** `launch diagnose` сопоставляет ошибку `xcodebuild`/Gradle/CocoaPods с причиной и исправлением
  на понятном языке; `launch fingerprint` показывает, почему следующая сборка будет чистой, а не инкрементальной.
  `--verbose` транслирует сырой вывод сборки вместо спиннера прогресса.

**Онбординг и обучение**

- **`launch demo`.** Симулированный обзор всего конвейера без настройки (проигрывается автоматически один раз при
  первом запуске); `--explain` к любой команде и `launch explain <topic>` раскрывают терминологию по требованию.

## Launch против EAS

Launch выполняет тот же конвейер `eas build` → `eas submit` → `eas update` (плюс `eas metadata` и
`eas credentials`) на оборудовании, которым вы владеете — и покрывает шаги настройки магазина, которые EAS оставляет
вам. Там, где двое расходятся в одном и том же рабочем процессе:

| Сборки **встают в очередь** в общем облаке, иногда на часы                                                                      | Сборки **запускаются немедленно** на вашем оборудовании — без очереди                                                                                            |
| Сборки бесплатного тарифа **ограничены 45-минутным таймаутом**                                                                  | **Без таймаута** — сборка идёт столько, сколько нужно                                                                                                            |
| Запросы **2FA** Apple ID / просроченные коды прерывают сборки                                                                   | Аутентифицируется по **ключу API App Store Connect (JWT)** — без пароля, без 2FA                                                                                 |
| Инструментарий/Node **закреплены образом сборки**; локальный `.env` не разрешается                                              | Ваш собственный Xcode/Node/инструментарий и документированная **лестница приоритета окружения** (`--print-env` для аудита)                                       |
| **Встроенные покупки и подписки** — ручная работа в UI ASC                                                                      | `launch sync` сверяет **IAP, подписки и возможности** из `launch.config.ts`                                                                                      |

| `eas metadata` работает **только с iOS**                                                                                        | `launch metadata` синхронизирует карточку для **iOS _и_ Android**                                                                                                |
| **IAP, подписки, треки и отзывы** Play означают UI Play Console                                                                 | `launch play-products` / `play-subscriptions` сверяют Play из **той же** конфигурации; `launch play-tracks` / `play-reviews` управляют треками и ответами по API |

## Требования

- **iOS:** членство в **Apple Developer Program** ($99/год) — [зарегистрируйтесь здесь](https://developer.apple.com/programs/enroll/) — затем macOS с **Xcode** + инструменты командной строки, **fastlane** (`brew install fastlane`) и
  **ключ API App Store Connect** (`.p8` + Key ID + Issuer ID) — [сгенерируйте его здесь](https://appstoreconnect.apple.com/access/integrations/api).
  Нет Mac? См. [Сборка без Mac](#сборка-без-mac).
- **Android:** **аккаунт разработчика Google Play** (единоразово $25) — [зарегистрируйтесь здесь](https://play.google.com/console/signup) — затем **JDK** (любая ОС — Mac не нужен) и JSON-ключ **сервисного аккаунта Google Play**.
- **Node 20+** на каждой платформе.

Запускайте `launch doctor` в любой момент, чтобы проверить всё вышеперечисленное.

## Быстрый старт

От пейволла до тестового трека за пять команд (замените `ios` → `android` для Google Play):

```bash
launch init                 # scaffold launch.config.ts + .env.example, tailored to your repo
launch creds set-key        # import your store API key into the OS keychain
launch creds setup          # register the app id + create/reuse signing assets
launch build ios --dry-run  # rehearse the whole flow — no network, no build, no account changes
launch build ios            # build, sign, size-check, and upload to the testing track
```

`launch build` молча переиспользует ваши кешированные учётные данные; если их нет, он предлагает подготовить их
по ходу. Публичный релиз — это отдельная, осознанная `launch release <platform>`.

Продаёте встроенные покупки или подписки? Объявите их в `launch.config.ts` и запустите `launch sync`, чтобы создать
и сверить их в App Store Connect — без кликов по порталу.

Уже публикуете приложение? `launch adopt` читает вашу действующую настройку App Store Connect — товары, возможности,
подпись и карточку — и записывает её обратно в конфиг за один шаг, чтобы вы могли продвигать её дальше через `sync`.

## Конфигурация

Факты о приложении (bundle id, версия) читаются из конфигурации Expo каждого приложения — `app.json` или
`app.config.{ts,js}` — так что они никогда не дублируются. `launch.config.ts` содержит только настройки, специфичные
для Launch:

```ts
import { defineConfig } from "launch-store";

export default defineConfig({
  // appRoots: ["./apps"],   // for a monorepo; omit to scan the repo root
  credentials: "local", // OS keychain + ~/.launch
  storage: "local", // ~/.launch/artifacts (swap for s3/r2/supabase later)
  buildEngine: "fastlane", // "fastlane" (local Mac) · "remote-mac" (AWS EC2 Mac) · "eas" (Expo cloud)
  // submit: "app-store-connect", // or "eas" to submit through Expo

  // Only needed to build iOS without a Mac via `--remote aws` — see "Building without a Mac".
  // aws: { region: "us-east-1" },

  profiles: {
    // `env` is inline per-profile vars; `envFile` renames the base dotenv. Precedence, highest first:
    // --env flags › keychain secrets › profile `env:` › .env.local (--include-local) › .env.<profile> › .env
    production: { name: "production", envFile: ".env", env: {}, sizeBudgetMB: 200 },
  },

  // Ping a Slack/Discord webhook and/or run a shell hook when a build or submit finishes (success or
  // failure). Both fields are optional; omit `notify` entirely for no notifications.
  // notify: { webhookUrl: "https://hooks.slack.com/services/…", command: "say build done" },

  // In-app purchases & subscriptions, keyed by bundle id — `launch sync` reconciles these onto App Store
  // Connect (and `launch play-products` / `play-subscriptions` onto Google Play). Capabilities aren't
  // declared here; they're read from app.json's `ios.entitlements`. Omit if your app sells nothing.
  // products: { "com.company.app": { subscriptionGroups: [/* … */], inAppPurchases: [/* … */] } },

  // Launch-native App Store Connect sections — each reconciled by its own command, declared inline here
  // (or, for back-compat, as a standalone `*.config.json` sidecar). Per-app ones are keyed by iOS bundle
  // id; Wallet & EU distribution are team-level. See examples/hello-world for a worked copy of each.
  // gameCenter: { "com.company.app": { achievements: [/* … */], leaderboards: [/* … */] } },
  // appClips: { "com.company.app": { clips: { "com.company.app.Clip": { action: "OPEN" } } } },
  // releaseAttributes: { "com.company.app": { pricing: { customerPrice: 9.99 }, categories: { primary: "PRODUCTIVITY" } } },
  // wallet: { merchantIds: [/* … */], passTypeIds: [/* … */] },
  // euDistribution: { domains: [/* … */] },
});
```

Запустите `launch build <platform> --print-env`, чтобы увидеть полностью разрешённое окружение и откуда пришло каждое
значение (секреты замаскированы), ещё до того, как запустится хоть одна сборка. Полнофункциональный пример для двух
платформ (iOS + Android) — задействующий каждую поверхность конфигурации-как-код, от товаров, предложений и атрибутов
релиза до Game Center, Wallet, распространения в ЕС и каталога Google Play — лежит в
[`examples/hello-world`](./examples/hello-world) (см. его README для пофункционального обзора).

## Сборка без Mac

Подпись iOS возможна только на macOS, так что разработчику на Windows/Linux где-то нужен Mac. Запустите `launch`
(мастер) или выберите путь напрямую. Android собирается везде, где работает JDK, поэтому к нему ничего из этого не
относится.

| Путь                     | Что происходит                                                                                                                  | Стоимость                                                                                                           |
| ------------------------ | ------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------- |
| **Облачный Mac AWS**     | Launch подготавливает EC2 Mac в **вашем собственном** аккаунте AWS, собирает + подписывает + отправляет, затем сворачивает его. | Вы платите AWS напрямую — **~$16 минимум за 24-часовую сессию** (лицензия Apple задаёт жёсткий 24-часовой минимум). |
| **Подключить Mac (SSH)** | Сборка на любом доступном вам Mac — коллеги, MacStadium, вручную запущенный инстанс.                                            | Сколько бы этот Mac вам ни стоил.                                                                                   |
| **Expo EAS**             | Launch оркеструет `eas-cli` из конца в конец (`eas build` → загрузка → `eas submit`) в облаке Expo.                             | **Бесплатный тариф** Expo с месячными лимитами.                                                                     |

```bash
launch build ios --remote aws            # build on a cloud Mac in your AWS account
launch build ios --remote ec2-user@host  # build on a Mac you reach over SSH
launch cloud doctor                      # check AWS creds, region, Mac-host quota, IAM
launch cloud status                      # live host: age, cost so far, releasable-after time
launch cloud teardown                    # stop + release the host (warns about the 24h floor)
```

Удалённые сборки загружают временную копию ваших ключей подписи на **ваш собственный** хост с явного согласия и
уничтожают их после — никогда на чужие серверы. Для редких сборок iOS macOS-раннер GitHub Actions дешевле, чем EC2
Mac; ценность Launch здесь — автоматизация в вашем собственном аккаунте с одними и теми же ключами везде.

## Как обрабатываются ваши учётные данные

- Ключ API (`.p8`), приватный ключ распространения и ключ загрузки Android живут в вашем **keychain ОС**.
- Сертификат iOS дополнительно резервируется как защищённый паролем `.p12` в `~/.launch/credentials/`
  (chmod 600); пароль хранится в keychain, никогда рядом с файлом.
- Ваш приватный ключ генерируется локально — в Apple отправляется только CSR.
- Launch переиспользует существующий сертификат распространения вместо создания новых (магазины ограничивают их число).

## FAQ

**Что такое Launch?** Launch — это открытая, self-hosted альтернатива Expo EAS: он собирает, подписывает и публикует приложения Expo / React Native в TestFlight и Google Play прямо с вашей машины, с вашими ключами, без платы за каждую сборку. Он выполняет тот же конвейер build → submit → update, что и EAS, и добавляет шаги настройки магазина, которые EAS оставляет сайтам App Store Connect и Play Console — встроенные покупки, подписки, возможности и метаданные карточки — как код.

**Является ли Launch бесплатной, открытой альтернативой Expo EAS?** Да. Launch распространяется под лицензией MIT и полностью с открытым исходным кодом, а сборки запускаются на оборудовании, которым вы владеете, — поэтому нет платы за каждую сборку, счётчика минут или ежемесячной подписки — в отличие от платных тарифов EAS за $19–$199/mo плюс перерасход за сборки. Единственная опциональная стоимость — аренда облачного Mac, если вам нужно собирать iOS без собственного.

**Чем Launch отличается от Expo EAS?** EAS запускает ваши сборки в облаке Expo и хранит учётные данные, артефакты и OTA-обновления на серверах Expo. Launch выполняет идентичный конвейер на вашей собственной машине, хранит ключи подписи в вашем keychain ОС и сохраняет артефакты и OTA-обновления в вашем собственном бакете (S3 / R2 / Supabase) — а затем управляет настройкой магазина, которой EAS не касается (IAP, подписки, возможности и карточка iOS и Android), как кодом. Команды отображаются один к одному: `eas build` → `launch build`, `eas submit` → `launch release`, `eas update` → `launch update`, `eas metadata` → `launch metadata`, `eas credentials` → `launch creds`.

**Можно ли собирать iOS-приложения без Mac?** Подпись кода iOS и цепочка инструментов сборки работают только на macOS, поэтому Mac должен присутствовать в процессе — но он не обязательно должен быть вашим. Launch может подготовить облачный Mac в вашем собственном аккаунте AWS (EC2 Mac), собрать по SSH на любом доступном вам Mac или передать работу облаку Expo EAS. Android собирается везде, где работает JDK, — без Mac вовсе.

**Поддерживает ли Launch Android и Google Play?** Да. Launch собирает и подписывает Android-приложения и загружает их в Google Play, а также сверяет встроенные товары Play, подписки (базовые планы + предложения), треки релизов и ответы на отзывы из того же каталога `launch.config.ts`, который управляет App Store Connect — один источник истины для обоих магазинов.

**Поддерживает ли Launch OTA-обновления, как EAS Update?** Да. `launch update` публикует JS и ассет-обновления по протоколу Expo Updates, на котором ваш рантайм `expo-updates` уже говорит — с подписью кода и хостингом в вашем собственном бакете (S3 / R2 / Supabase), а не на серверах Expo. `launch updates rollback` отменяет неудачный релиз, продвигая заведомо рабочее обновление или возвращая клиентов на встроенный бандл.

**Как мигрировать с Expo EAS на Launch?** Замените команды один к одному (`eas build` → `launch build`, `eas submit` → `launch release`, `eas update` → `launch update`, `eas credentials` → `launch creds`, `eas metadata` → `launch metadata`). Если ваше приложение уже публикуется, `launch adopt` читает вашу действующую настройку App Store Connect — товары, возможности, подпись и карточку — и записывает её обратно в `launch.config.ts` за один шаг. Launch также может по-прежнему передавать работу `eas build`, когда у вас нет Mac, — так что можно мигрировать постепенно.

**Launch — это просто SDK для App Store Connect или MCP-обёртка?** Нет. SDK или MCP-сервер App Store Connect оборачивает лишь часть API Apple. Launch управляет всем релизом сразу в Apple и Google — подпись кода, нативные сборки, проверка размера, настройка магазина как код, подтверждённый публичный релиз и OTA-обновления — ничего из этого API-обёртка не касается. Если вам нужен self-hosted Expo EAS, а не клиент API — это Launch.

**Чем Launch отличается от Fastlane?** Fastlane — строительный блок; Launch оркеструет его. Launch использует fastlane только для шага загрузки бинарника и выстраивает вокруг него весь релиз: подготовку учётных данных, сборку, реальную проверку размера загрузки, настройку магазина как код для обоих магазинов, осознанный публичный релиз, контроль поэтапного выкатывания и OTA-обновления — всё из одного типизированного `launch.config.ts`.

**Где хранятся мои ключи подписи и секреты?** В вашем keychain ОС. Ваш ключ API App Store Connect (`.p8`), приватный ключ распространения и ключ загрузки Android никогда не касаются репозитория или чьих-либо серверов — в Apple отправляется только запрос на подпись сертификата (CSR). Секреты сборки также хранятся в keychain через `launch secret`, чтобы они не попадали в закоммиченный `.env`.

**Что нужно для запуска Launch?** Node 20+ везде. Для iOS: macOS с Xcode и инструментами командной строки, fastlane и ключ API App Store Connect (`.p8` + Key ID + Issuer ID) — или удалённый Mac, если у вас его нет. Для Android: JDK (любая ОС) и JSON-ключ сервисного аккаунта Google Play. Запустите `launch doctor`, чтобы проверить всё сразу.

**Сколько стоит Launch?** Launch сам по себе бесплатен (MIT). Вы платите только за то, что платили бы в любом случае: ваше собственное оборудование для сборки (или время облачного Mac, если собираете iOS без локального Mac), плюс обычные регистрационные взносы Apple Developer ($99/yr) и Google Play (единоразово $25). Нет платы за каждую сборку и нет подписки.

**Работает ли Launch в CI?** Да. `launch ci init` создаёт каркас рабочего процесса GitHub Actions на macOS-раннере, и каждая команда переходит в неинтерактивный режим при обнаружении CI, пайпа или агента — так что один и тот же поток работает без присмотра.

**Какие фреймворки поддерживает Launch?** Приложения Expo и bare React Native, которые описывают себя через конфигурацию Expo (`app.json` / `app.config.{ts,js}`) и `expo prebuild`. Launch читает ваш bundle id, версию и entitlements оттуда, поэтому ничего не дублируется.

**Может ли Launch управлять метаданными магазина, встроенными покупками и подписками?** Да — как кодом, для обоих магазинов. `launch sync` сверяет IAP, подписки, цены, возможности и локализованную карточку (тексты, скриншоты, превью) с App Store Connect; `launch metadata` покрывает карточку для iOS и Android; `launch play-products` / `launch play-subscriptions` управляют каталогом Google Play. Каждая команда выполняет план только для чтения → подтверждение → применение, поэтому никогда не затирает живую или находящуюся на ревью версию.

**Привязывает ли Launch меня к хостируемому сервису?** Нет — нет ничего хостируемого и ничего проприетарного. Launch распространяется под лицензией MIT, построен на fastlane, Gradle и собственных API платформ, с подключаемыми провайдерами хранилища / учётных данных / сборки / отправки. Ваши ключи, артефакты и обновления хранятся в инфраструктуре под вашим контролем, так что мигрировать потом не с чего.

**Как начать?** Установите командой `npm install --global launch-store` (или `--save-dev` на проект), затем запустите `launch demo` для 60-секундного симулированного обзора — без настройки и аккаунта. Когда будете готовы: `launch init` → `launch creds set-key` → `launch creds setup` → `launch build ios`.
